# 대광테크 관리자 사용 가이드

## 1. 접속 주소

- 관리자: https://dgtc.ejdzm90.workers.dev/#/admin
- 공개 홈페이지: https://dgtc.ejdzm90.workers.dev/#/
- 공지사항: https://dgtc.ejdzm90.workers.dev/#/notice

## 2. 로그인 방법

1. 관리자 주소로 접속한다.
2. 중앙 로그인 창에서 아이디와 비밀번호를 입력한다.
3. 로그인 버튼을 누르거나 Enter 키를 누른다.

현재 데모 로그인:

- ID: dgtc66
- PW: 1234

현재 로그인은 프론트 데모 게이트다. 실제 운영 전에는 서버 세션 인증으로 교체해야 한다.

## 3. 로그아웃 방법

관리자 콘솔 우측 상단의 `로그아웃` 버튼을 누른다. 확인 창에서 확인을 누르면 다시 로그인 화면으로 돌아간다.

## 4. 대시보드 사용법

대시보드는 이미지, 공지사항, 최근 활동, 백업 상태를 빠르게 확인하는 첫 화면이다. 요약 카드의 `자세히 보기`나 빠른 작업 버튼을 누르면 해당 관리 섹션으로 이동한다.

## 5. 이미지 관리 사용법

- 메인 배너, 회사소개, 설비 이미지, 제품 이미지 탭을 선택한다.
- 업로드 영역을 눌러 JPG/PNG/WEBP 이미지를 추가한다.
- 이미지를 선택하면 우측 미리보기 패널에서 파일명, 크기, 해상도, 사용 여부를 확인할 수 있다.
- 교체, 삭제, 순서 이동, 제목 변경은 이미지 카드와 편집 영역에서 처리한다.

주의: 현재 이미지는 같은 브라우저의 localStorage 기준으로 저장된다. 모든 방문자에게 같은 이미지가 보이려면 R2 업로드와 서버 저장 연동이 필요하다.

## 6. 공지사항 관리 사용법

- 제목, 템플릿, 카테고리, 게시일, 노출 여부, 중요 공지 여부, 내용을 입력한다.
- `공지 등록`을 누르면 목록에 추가되고 공개 공지사항 페이지에도 반영된다.
- `미리보기`로 공개 화면에 가까운 내용을 확인한다.
- 목록에서 눈 아이콘은 미리보기, SEO 아이콘은 공개 보기, 연필 아이콘은 수정, 휴지통 아이콘은 삭제다.
- 비노출로 바꾸면 공개 목록과 상세 페이지에서 보이지 않는다.

## 7. 공지사항 공개 페이지 확인법

- 목록: https://dgtc.ejdzm90.workers.dev/#/notice
- 상세: 공지 목록의 `자세히 보기`를 누른다.
- 관리자에서 새 공지를 등록한 뒤 같은 브라우저에서 공개 페이지를 새로고침하면 반영 상태를 확인할 수 있다.

## 8. 메뉴 관리 사용법

메뉴명, 경로, 노출 여부, 순서를 관리한다. 새 메뉴를 추가할 때 경로는 `/notice`처럼 `/`로 시작해야 한다.

현재 단계에서는 관리자 메뉴 데이터가 localStorage에 저장된다. 모든 방문자 공통 메뉴 반영은 서버 저장 전환 후 적용한다.

## 9. 푸터 관리 사용법

회사명, 대표자, 사업자등록번호, 주소, 대표전화, 이메일, 저작권 문구를 수정하고 저장한다. 저장 후 같은 브라우저 기준으로 유지된다.

## 10. SEO 설정 사용법

메타 타이틀, 설명, 키워드, OG 이미지명, robots 설정을 관리한다. 미리보기 카드에서 검색 노출 문구를 확인할 수 있다.

## 11. 백업 관리 사용법

- `백업 생성`을 누르면 현재 관리자 데이터를 JSON으로 내려받고 백업 목록에 추가한다.
- 다운로드 버튼은 백업 파일을 다시 내려받는다.
- 복원 버튼은 선택한 백업의 스냅샷이 있을 때 관리자 상태를 복원한다.

## 12. 주의사항

- 현재 저장은 같은 브라우저 기준이다.
- 브라우저 localStorage를 삭제하면 관리자 데이터가 초기화될 수 있다.
- 실제 운영 전에는 서버 인증, D1 저장, R2 이미지 업로드가 필요하다.
- 로그인 ID/PW는 프론트 번들에서 보호되지 않는 데모용이다.

## 13. 현재 localStorage 기반 한계

- 다른 사용자 브라우저에는 동일 데이터가 자동 공유되지 않는다.
- 서버 권한 검증이 없다.
- 이미지 파일은 실제 R2 서버 저장이 아니다.
- 감사 로그도 서버 원장으로 봉인되지 않는다.

## 14. 다음 운영화 예정 항목

- Worker 서버 인증/세션
- D1 공지/메뉴/푸터/SEO/설정 저장
- R2 이미지 업로드
- Public API 공통 데이터 반영
- 권한별 관리자 보호
- 서버 감사 로그
